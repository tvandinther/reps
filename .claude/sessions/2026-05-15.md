## Non-obvious decisions

**Gradle wrapper binary downloaded via curl**
`gradle-wrapper.jar` is a binary that can't be generated without Gradle installed. Downloaded from `github.com/gradle/gradle/raw/v8.9.0/...`. If Gradle version is bumped later, re-download the jar to match.

**Kotlin 2.0.21 + `kotlin.plugin.compose`**
With Kotlin 2.0+, the Compose compiler is no longer a separate artifact. The `org.jetbrains.kotlin.plugin.compose` plugin handles it — no `composeOptions.kotlinCompilerExtensionVersion` needed. This is the modern approach as of Kotlin 2.0.

**KSP over KAPT for Room**
KAPT is deprecated in Kotlin 2.0. KSP (`com.google.devtools.ksp`) is the correct choice.

**Type-safe navigation routes via kotlinx-serialization**
Compose Navigation 2.8+ supports type-safe routes using `@Serializable` data objects/classes. Added `kotlin.plugin.serialization` and `kotlinx-serialization-json` to enable this. Routes: `ExercisesTab`, `ExercisesList`, `SetLogging(exerciseId: Long)`, `HistoryTab`.

**SessionAssembler input ordering**
The DAO returns rows `ORDER BY logged_at DESC` (newest first). `session_break = 1` on a row means "the gap before this row (looking backwards in time) exceeds the threshold" — i.e., this row is the *most recent* set of the older session. The assembler splits on this flag while iterating DESC.

**LoggingViewModel computes sessions in-memory**
The `getSetsWithSessionBreaks` query is used for the History screen (all exercises). For the Set Logging screen, session clustering is done in the ViewModel over `getForExercise` results to avoid a second complex query scoped to one exercise. This is simpler and correct for the single-exercise case.

**windowSoftInputMode="adjustResize"**
Set on the Activity in the manifest so the keyboard pushes the layout up rather than overlapping it. Combined with `imePadding()` in Compose, this keeps the search bar and "Add set" sheet above the keyboard.
